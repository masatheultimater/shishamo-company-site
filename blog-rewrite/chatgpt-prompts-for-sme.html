ChatGPT、登録はしましたか？

「登録はしたけど、結局使ってない」。正直、そういう会社がほとんどです。理由はシンプルで、何を聞けばいいかわからないから。

今回は「これだけコピペすればOK」のプロンプトを10個、実際に私が毎日の業務で使っているものから厳選しました。1つ試すだけで「おっ」となるはずです。

この記事でわかること

  • コピペですぐ使えるChatGPTプロンプト10個（営業・社内業務・経理）
  • ChatGPTで「使えない回答」が返ってくる原因と、たった3つのコツ
  • セキュリティ面で入れてはいけない情報の線引き

その前に：ChatGPTで「使えない回答」が返ってくる理由

「ChatGPTに聞いてみたけど、なんか微妙な回答だった」。これ、ChatGPTが悪いんじゃなくて、聞き方の問題です。

ChatGPTは超優秀だけど空気が読めない新入社員だと思ってください。「いい感じにやっといて」で動ける人間と違って、曖昧な指示には曖昧な答えしか返ってこない。当たり前の話なんですよね。

コツは3つだけ。

  1. 役割を与える —「あなたは中小企業の経理担当です」
  2. 具体的な条件を入れる — 業種、相手の立場、文書の目的
  3. 出力フォーマットを指定する — メール文面、箇条書き、表

この3つ、以降の10個のプロンプト全部に共通するパターンです。逆に言えば、この3つさえ押さえておけば、自分でアレンジしたプロンプトも格段に精度が上がります。

「プロンプトが上手い人」は、この3つを無意識にやっているだけ。 特別な才能じゃありません。

カテゴリA: 営業・顧客対応

1. 営業メールの下書き

こんな時に使う: 新規開拓やフォローアップのメールを書くとき 時短効果: 30分 → 5分

あなたはBtoB営業のプロフェッショナルです。

以下の条件で、営業メールの下書きを作成してください。

【送信元】
- 会社名: [御社名]
- 業種: [業種]
- サービス内容: [提供しているサービスを1〜2行で]

【送信先】
- 会社名: [相手の会社名]
- 相手の役職: [相手の役職]
- 関係性: [初回アプローチ / 過去に名刺交換 / 既存顧客へのフォロー]

【メールの目的】
[例: 初回面談のアポイント獲得 / セミナー案内 / お礼と次回提案]

【トーン】
丁寧だが堅すぎない。相手の時間を奪わない簡潔さを重視。

【出力形式】
件名 + 本文（300字以内）

使い方のコツ: [関係性]を変えるだけで、初回・フォロー・お礼と使い分けられます。件名は3パターンくらい出してもらうと選びやすい。

2. 顧客への提案文作成

こんな時に使う: 相手の課題に合わせた提案書の骨子がほしいとき 時短効果: 2時間 → 20分

あなたは中小企業向けのコンサルタントです。

以下の情報をもとに、顧客への提案書の骨子を作成してください。

【提案先の情報】
- 業種: [相手の業種]
- 従業員規模: [おおよその人数]
- 相手が抱えている課題: [例: 紙の請求書処理に時間がかかっている]

【提案したいこと】
[例: クラウド請求書ツールの導入支援]

【提案書に含めてほしい項目】
1. 現状の課題（相手の視点で）
2. 提案内容の概要
3. 導入メリット（具体的な数字を仮置きで）
4. 導入スケジュール（ざっくり）
5. 費用感（概算でOK）

【トーン】
専門用語は避け、経営者が読んでわかる言葉で。A4で2ページ以内。

使い方のコツ: 出てきた文章をそのまま使うのではなく、「叩き台」として自分の言葉で直すのが大事。特に費用感は自社の実情に合わせて必ず修正してください。

3. クレーム対応メールの下書き

こんな時に使う: 感情的にならず、丁寧に返信したいとき 時短効果: 45分 → 10分

あなたはカスタマーサポートのベテラン担当者です。

以下のクレーム内容に対する返信メールを作成してください。

【クレームの概要】
[例: 納品された商品の一部に不良品が混在していた]

【お客様の感情の温度感】
[かなり怒っている / 冷静だが不満 / 困っている]

【こちらの対応方針】
[例: 不良品を即日交換。送料はこちら負担。原因調査中]

【メールに含めてほしい要素】
1. お詫びの言葉（最初に）
2. 事実確認の内容
3. 具体的な対応策と期限
4. 再発防止への言及
5. 感謝の言葉（最後に）

【トーン】
誠実で丁寧。言い訳は入れない。300字以内。

使い方のコツ: クレーム対応は感情が入ると文章がブレがち。一度ChatGPTに冷静な叩き台を出してもらってから、自分の言葉を足すのがおすすめです。

正直に言うと、この営業カテゴリの3つは「まあ便利だよね」くらいの感覚。ChatGPTの本領は次のカテゴリからです。個人的に一番使っているのは、この後の#4（議事録）と#9（Excel関数）。

カテゴリB: 社内業務

4. 議事録の要約

こんな時に使う: 会議の録音テキストやメモから、要点だけ抜き出したいとき 時短効果: 1時間 → 5分

あなたは議事録作成のプロフェッショナルです。

以下の会議メモ（または録音の文字起こし）を、構造化された議事録に要約してください。

【会議の基本情報】
- 会議名: [例: 月次営業会議]
- 日時: [日時]
- 参加者: [参加者名]

【会議メモ・文字起こし】
ここに会議メモを貼り付けてください。多少雑でも大丈夫です。
---
[ここに貼り付け]
---

【出力フォーマット】
1. 議題一覧（箇条書き）
2. 各議題の要点（それぞれ2〜3行）
3. 決定事項（明確に）
4. TODO（担当者・期限を明記）
5. 次回までの宿題

使い方のコツ: 文字起こしは多少ぐちゃぐちゃでも大丈夫です。ZoomやTeamsの自動文字起こしをそのまま貼り付けても、かなりの精度で整理してくれる。これだけでChatGPTの月額3,000円の元が取れます。 冗談じゃなく本気でそう思っています。

5. 社内通知・お知らせ文

こんな時に使う: 制度変更やイベント告知を社内に周知するとき 時短効果: 20分 → 3分

あなたは中小企業の総務担当です。

以下の内容で、社内向けのお知らせ文を作成してください。

【お知らせの種類】
[制度変更 / イベント告知 / 業務連絡 / 注意喚起]

【伝えたい内容】
[例: 来月から経費精算の締日が毎月25日→20日に変更]

【対象者】
[全社員 / 管理職のみ / 特定部署]

【トーン】
社内メールとして自然な丁寧さ。堅すぎず、くだけすぎず。

【出力形式】
件名 + 本文（200字以内）。要点は箇条書きで。

使い方のコツ: 地味なプロンプトに見えるかもしれません。でも「社内通知を書くのに20分悩んでいる」人、思っている以上に多いんですよね。

6. マニュアル作成の叩き台

こんな時に使う: 業務手順を「誰でもわかる形」で文書化したいとき 時短効果: 半日 → 30分

あなたは業務改善コンサルタントです。

以下の業務について、新入社員でも理解できるマニュアルの叩き台を作成してください。

【業務名】
[例: 月末の請求書発行作業]

【業務の流れ（箇条書きでOK）】
[例:
- Excelの売上一覧を確認
- 請求書テンプレートに転記
- 上長に確認依頼
- PDFにしてメール送付
- 送付済みフォルダに移動]

【使用ツール】
[例: Excel, Gmail, Google Drive]

【想定する読み手】
[例: 経理経験のない新入社員]

【出力形式】
- 手順は番号付きで
- 各手順に「注意点」を1行追加
- 所要時間の目安も記載

使い方のコツ: 完璧なマニュアルを一発で作ろうとしないこと。まず叩き台を出して、実際にやっている人に「ここ違うよ」を赤入れしてもらうのが一番早いです。

こんな時に使う: 求人サイトや自社HPに載せる募集文を書くとき 時短効果: 1時間 → 15分

あなたは中小企業の採用担当です。

以下の条件で、求職者に魅力が伝わる求人原稿を作成してください。

【募集職種】
[例: 経理事務スタッフ]

【雇用形態】
[正社員 / パート / 契約社員]

【仕事内容】
[例: 月次の仕訳入力、請求書処理、経費精算チェック]

【求める人物像】
[例: Excel基本操作ができる方。経理経験は不問]

【会社の魅力・アピールポイント】
[例: 残業月10時間以下、少人数でアットホーム、駅徒歩5分]

【給与・勤務条件】

【出力形式】

- 応募資格

使い方のコツ: 「先輩社員の声」は架空なので、実際の社員に聞いた話に差し替えるのがおすすめ。#7は正直、自分で書いたほうが味が出る場合もあります。でも「何も書けなくて止まっている」よりは、叩き台があるほうが100倍マシ。

ここまで7つ。「ChatGPTって文章生成でしょ？」と思っていた方、次の3つで印象が変わるかもしれません。

カテゴリC: 経理・管理

8. 経費精算ルールの整理

こんな時に使う: 曖昧なまま運用されている社内ルールを明文化したいとき 時短効果: 3時間 → 30分

あなたは中小企業の管理部門のマネージャーです。

以下の情報をもとに、社員向けの経費精算ルールを整理してください。

【現在のルール（わかっている範囲で）】
[例:
- 交通費は実費精算
- 接待は事前承認が必要（たぶん）
- 領収書は原本提出（コピー不可？）
- 締日は毎月25日]

【曖昧な点・困っていること】
[例:
- タクシー利用の基準が不明確
- 少額（1,000円以下）の扱いが人による
- 領収書を紛失した場合の対応が決まっていない]

【出力形式】
1. 経費の種類ごとのルール（表形式）
2. 申請フロー（ステップ形式）
3. Q&A（よくある質問5つ）
4. 「これはNG」の具体例

使い方のコツ: 「曖昧な点」を正直に書くほど、実用的なアウトプットが出ます。完璧に把握していなくても大丈夫。むしろ「たぶんこうだけど自信がない」を洗い出す作業自体に価値があります。

9. Excel関数の作成

こんな時に使う: 「やりたいこと」はあるのに、関数がわからないとき 時短効果: 30分 → 2分

あなたはExcelのエキスパートです。

以下の「やりたいこと」を実現するExcel関数を教えてください。

【やりたいこと】
[例: A列に日付、B列に売上金額が入っている。C1に「2024年3月」の売上合計を出したい]

【シートの構成】
[例: A列＝日付（2024/1/1〜2024/12/31）、B列＝金額（数値）]

【Excel のバージョン】
[Excel 2019 / Microsoft 365 / わからない]

【出力形式】
1. 関数（コピペ用）
2. 関数の意味を1行で解説
3. 応用例を1つ

使い方のコツ: 「やりたいこと」は日本語でざっくり書けばOKです。VLOOKUPで30分悩んでいたのが2分で解決する。経理の方にこれを見せたら「なにこれ魔法？」と言われました。嘘みたいな本当の話。

こんな時に使う: 毎月の経営報告を文章にまとめるとき 時短効果: 1時間 → 10分

あなたは中小企業の経営管理担当です。

以下の数値データをもとに、経営者向けの月次報告書の骨子を作成してください。

【対象月】
[例: 2024年12月]

【主な数値】
- 売上: [金額]（前月比: [+○%/-○%]）
- 営業利益: [金額]（前月比: [+○%/-○%]）
- 主な増減要因: [例: 新規案件2件獲得、外注費が増加]

【特記事項】
[例: 大口顧客との契約更新あり、来月から新サービス開始予定]

【トーン】

使い方のコツ: 数字を入れるだけで、「今月のサマリー」が自動で文章になります。報告書の「型」を毎月ゼロから考える必要がなくなった。

「でもセキュリティが心配...」への回答

この不安、正しいです。むしろ何も心配せずに使うほうが危険。

ChatGPTに入れてはいけない情報は3つ。

  1. 個人情報 — 顧客名、住所、電話番号、マイナンバー
  2. 契約金額 — 見積書の具体的な金額、取引条件
  3. 未公開情報 — 新製品の開発内容、M&A情報、未発表の人事

逆に言えば、それ以外の定型業務なら問題ありません。今回紹介した10個のプロンプトは、すべて「機密情報を入力しなくても使える」ものを選んでいます。

もう一つ。ChatGPT有料版のTeamプラン（月額約4,200円/人）以上なら、入力データがAIの学習に使われない設定になっています。本格的に全社導入するなら、このプランが最低ラインです。

「とりあえず、社外秘じゃない定型業務から始める」のがおすすめです。

まとめ

  • ChatGPTに上手く聞くコツは「役割・条件・出力形式」の3つだけ
  • まずは議事録の要約（#4）かExcel関数（#9）を試すのがおすすめ
  • 機密情報は入れない。社外秘じゃない定型業務から始めればOK

10個のプロンプト、全部使う必要はありません。まずは1つだけ、明日の朝に試してみてください。5分で「これは使える」と感じるか、「うちにはまだ早い」と感じるか。それがわかるだけでも収穫です。

ただ、もし「もっと自社の業務に合わせたプロンプトがほしい」「全社で使うルールを整備したい」と感じたら、それは次のステップのサイン。

業務フロー全体を見てプロンプトを設計する。社内の利用ガイドラインを整備する。御社専用のAIツールを構築する。そこまで踏み込むのが、私の生成AI導入支援とAIツール開発です。

「ChatGPTの使い方、もう少し踏み込んで相談したい」。そんな方は、30分の無料相談からどうぞ。雑談くらいの気持ちで構いません。
